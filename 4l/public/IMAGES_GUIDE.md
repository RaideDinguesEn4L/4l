# Guide des Images Locales

## 📁 Structure des dossiers

```
public/
└── images/
    ├── hero/
    │   └── desert-hero.jpg          # Image principale hero (1920x1080px min)
    ├── team/
    │   ├── thomas.jpg               # Photo de Thomas (400x400px)
    │   └── marie.jpg                # Photo de Marie (400x400px)
    ├── car/
    │   ├── 4l-main.jpg              # Photo principale de la 4L (800x600px)
    │   └── desert-background.jpg    # Fond section Trophy (1920x1080px)
    └── blog/
        ├── acquisition.jpg          # Article "4L trouvée" (600x400px)
        ├── mecanique.jpg            # Article "Préparation" (600x400px)
        ├── partenaires.jpg          # Article "Partenaires" (600x400px)
        └── depart.jpg               # Article "Départ" (600x400px)
```

## 📸 Images à préparer

### 1. Hero (desert-hero.jpg)
- **Taille recommandée** : 1920x1080px minimum
- **Format** : JPG
- **Description** : Paysage désertique au coucher du soleil avec dunes

### 2. Équipe
- **thomas.jpg** : Photo de Thomas (pilote)
- **marie.jpg** : Photo de Marie (co-pilote)
- **Taille** : 400x400px (carré)
- **Format** : JPG

### 3. 4L & Préparation
- **4l-main.jpg** : Photo de votre Renault 4L
- **Taille** : 800x600px
- **Format** : JPG

### 4. Trophy Background
- **desert-background.jpg** : Fond pour section Trophy
- **Taille** : 1920x1080px
- **Format** : JPG

### 5. Articles
- **acquisition.jpg** : Photo de l'achat de la 4L
- **mecanique.jpg** : Photo de la préparation mécanique
- **partenaires.jpg** : Photo d'événement ou logo
- **depart.jpg** : Photo du désert/départ
- **Taille** : 600x400px chacune
- **Format** : JPG

## ⚙️ Optimisation des images

Avant de placer les images, optimisez-les :
- Réduisez la taille de fichier (compression JPEG 80-85%)
- Utilisez des dimensions adaptées
- Privilégiez le format JPEG pour les photos
- Pour les logos : PNG avec transparence

## 🚀 Mise en place

1. Créez les dossiers dans `public/images/`
2. Placez vos images avec les noms indiqués
3. Les composants sont déjà configurés pour les utiliser
4. Les images seront accessibles via `/images/...`
